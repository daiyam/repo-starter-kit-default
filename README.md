Purpose
-------

Opinionated repository starter kit providing a small, useful set of defaults for new projects:

- GitHub issue labels and colors (`configs/labels.yml`)
- Issue/Discussion templates (`configs/`)

Labels summary
--------------

Labels are defined in `configs/labels.yml` and grouped by prefix:

- `S-*` (Status): lifecycle states (e.g. `S-in-progress`, `S-needs-triage`, `S-released`).
- `P-*` (Priority): `P-critical`, `P-high`, `P-medium`, `P-low`.
- `T-*` (Type): `T-bug`, `T-feature`, `T-improvement`, etc.
- `A-*` (Area): areas like `A-ci`, `A-docs`, `A-test`.
- `M-*` (Meta): community/process labels such as `M-help-wanted`.
